# BTC 數據分析與交易策略平台 (專案名稱)
   
## 1. 專案概覽
 
本專案旨在建立一個全面的比特幣（BTC）數據平台。初期目標是抓取、儲存和分析歷史價格數據。未來將擴展至實現交易策略回測、即時數據監控，並最終可能接入實盤交易。

## 2. 專案目標

**階段一：數據基礎設施 (目前階段)**
[ ] 抓取近三年的 BTC/USDT 每分鐘 K 線數據。
[ ] 抓取近三年的 BTC/USDT 每分鐘 交易量數據。
[ ] 將數據儲存為 npz 檔案。


**階段二：數據分析與指標**
[ ] 實現常見技術指標的計算（例如：移動平均線 MA, KD,相對強弱指數 RSI）。
[ ] 提供數據可視化功能。

**階段三：策略回測**
[ ] 建立一個策略回測框架。
[ ] 實現一個簡單的移動平均線交叉策略作為範例。

## 3. 技術棧與架構

**語言**: Python 3.10+
**核心框架**: 無 (純 Python 腳本)
**主要函式庫**:
`python-binance`: 數據來源
`pandas`: 數據處理
`numpy`: 數值計算
**架構原則**:
**嚴格遵守 SOLID 原則**。
所有新功能都應以模組化、可測試的方式開發。
清晰的職責分離（數據層、邏輯層、表現層）。
使用git做版本控制, 將每個步驟都使用標準commit紀錄, 自行push到 git@github.com:sky14785214/Binance_bot.git


## 4. 核心模組說明

**`config.py`**: 存放所有全域設定。
**`data_fetcher.py`**: 負責從外部 API (如幣安) 獲取數據。
**`data_processor.py`**: 負責數據的清洗、轉換和格式化。
**`data_saver.py`**: 負責將處理後的數據寫入儲存系統 (目前是 CSV)。
**`main.py`**: 應用程式的進入點與總協調器。

## 5. 開發與執行指南

**環境設定**:
```bash
pip install -r requirements.txt
```

## 6. 開發規範與偏好

**程式碼風格**: 遵循 [PEP 8](https://www.python.org/dev/peps/pep-0008/) 風格指南。
**註解**: 為所有公開的函式和類別編寫清晰的 Docstrings。對於複雜的邏輯區塊，請添加行內註解。
**日誌**: 未來應引入 `logging` 模組來取代 `print()` 語句，以便更好地進行日誌管理。
**錯誤處理**: 必須對所有外部 I/O（網路請求、檔案讀寫）和可能的計算錯誤進行穩健的錯誤處理。
